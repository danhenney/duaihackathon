# 노하우 및 프로젝트 진행 과정

## 진행 과정

1. paste.trade와 크립토 PNL 카드 UX를 참고해 검색, 피드, 랭킹, 마이 정보 탭 구조를 정했습니다.
2. 초기에는 라이브 검색을 붙였지만 해커톤 데모 안정성을 위해 저장된 검증 데이터 중심 검색으로 방향을 바꿨습니다.
3. 인플루언서, 종목, 공개 의견, 가격 차트 데이터를 분리해 검색과 상세 화면이 같은 데이터를 재사용하도록 만들었습니다.
4. Supabase/Postgres 연결을 준비해 향후 검증 의견이 쌓이면 시드 데이터와 DB 데이터가 함께 로드되도록 구성했습니다.
5. 세일러처럼 공개 발언 빈도가 높은 인물은 실제 X 원문을 추가해 반복 의견이 누락되지 않도록 보강했습니다.

## 노하우

- 라이브 수집보다 데모에서는 검증된 원문 데이터의 정합성이 더 중요했습니다.
- 없는 종목도 추적 유니버스에는 포함하되, 성과 랭킹은 출처가 확인된 의견만 반영해야 신뢰도가 유지됩니다.
- 인물 상세와 종목 상세는 PNL 카드보다 원문 URL, 발언 요약, 의견 시점이 먼저 보여야 사용자가 바로 납득할 수 있습니다.
- 투자 앱처럼 보이되 투자 조언이 아니라 공개 의견의 성과 기록이라는 톤을 유지하는 것이 중요했습니다.

## 다음 단계

- Supabase에 검증 의견 입력용 어드민 테이블을 만들고 운영자가 원문 URL을 추가하면 자동으로 앱에 반영되게 합니다.
- YouTube transcript와 Telegram 채널도 수동 검증 큐에 넣어 한국 인플루언서와 크립토 채널 커버리지를 넓힙니다.
- 인물별 공유 카드와 종목별 최초 포착자 랭킹을 더 시각적으로 강화합니다.
